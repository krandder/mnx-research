MNX Proposal

Funding design for perpetual markets

Recommendation: keep MNX's current funding-index settlement engine, replace the weak trade-clearing-price rate input with deterministic impact-depth samples, and fix session-boundary funding toggles before U.S. equity markets carry real money.

Recommendation Formula Equity sessions Contracts Liquidation and ADL Prior art Implementation

Concrete recommendation

Do not redesign MNX funding settlement. Keep the current Perpetual.globalIndex, account-local PackedAccountState.fundingIndex, _loadContext(true), _settlePendingFunding, and settleFundingForAccounts model. The change is the rate input: stop deriving normal funding from one batch clearing price or a broad setOffChainFundingRate override. Compute FundingOracle.currentFundingRate from deterministic one-minute impact-depth samples against the contract oracle price.

Window 1 hour Keep FUNDING_RATE_WINDOW.
Normal rate input 1 depth sample / minute Replace clearing-price samples.
Funding cap +/- 5 bps / hour 10 bps only for approved liquid markets.

Funding formula

Use the impact-bid / impact-ask premium used by QFEX, dYdX, Binance, and related venues, but normalize it to MNX's one-hour window and conservative launch caps.

raw_premium = (max(impact_bid - index_price, 0) - max(index_price - impact_ask, 0)) / index_price sample_premium = clamp(raw_premium, -maxFundingPremium, maxFundingPremium) window_premium = time_weighted_average(sample_premium over active seconds) funding_rate_window = clamp(window_premium / 24, -maxFundingRatePerWindow, +maxFundingRatePerWindow)

If impact_bid <= index_price <= impact_ask, funding is zero. A normal spread around the fair index should not create funding; persistent directional pressure should.

U.S. equity sessions

Launch rule

For U.S.-listed equities and indexes, MNX v1 should accrue funding only during the regular U.S. equity session: 9:30 AM to 4:00 PM New York time on trading days, with holidays and early closes respected. Overnight, weekends, and holidays must record zero-accrual funding heartbeats, not smear Friday premium across closed-market time.

Trading outside session

Trading may remain available with after-hours reference prices, but funding should be frozen unless MNX has a market-specific reference-source policy for that session. While frozen, tighten open-interest growth, require fresh risk marks, and allow cancels, reductions, margin adds, liquidation, and ADL.

Market type Funding session Reopen behavior
Crypto, FX, metals with continuous reference always Normal hourly windows.
U.S. equities and equity indexes us_equity_regular Settle to close boundary, freeze accrual, reset accumulator at next open.
Approved extended-hours equities us_equity_extended later only 4:00 AM to 8:00 PM ET after reference and depth policy is approved.

Contracts

The contract change should be small: preserve settlement, change the premium source, and make emergency overrides auditable.

Area Keep Change
State FundingOracle.currentFundingRate, currentFundingWindow, windowSet, Perpetual.globalIndex, and per-account fundingIndex. Add sample metadata only if needed for replay/audit: impact notional, depth-valid flag, config sequence, and sample timestamp. Do not add a second funding index.
Sample entrypoint Provider gating: today the provider is the Perpetual, and only it calls FundingOracle.recordTrade. Replace or extend recordTrade with recordImpactSample(impactBid, impactAsk, oraclePrice, timestamp, flags). The oracle price should still be the contract-accepted price, not a separate submitter-provided mark.
Window close Perpetual.setFundingRate() first calls _loadContext(true) to settle the old per-second rate into globalIndex, then calls FundingOracle.setFundingRate(). Keep that ordering. The rate update changes the next per-second rate; index accrual remains lazy through _loadContext(true).
Override path Keep an owner/operator emergency path for stuck markets. Treat setOffChainFundingRate as break-glass only: require a reason/event, alerting, max duration, and postmortem note. It should not be the ordinary funding-rate setter.

The normal production path should be reproducible from accepted impact samples, FundingRateUpdate events, and GlobalIndexUpdate events. A human-entered funding rate is an incident response tool, not market design.

Liquidation and ADL invariants

Settlement ordering

Preserve the current invariant: position-mutating contract paths apply funding before final account-state checks, and HTV1 preflight mirrors pending debits with computePendingFundingDebitRaw before accepting risk-increasing orders. Cancels, margin adds, closes, liquidation, and ADL must remain available even when settlement floors margin.

ADL capacity

Funding moves the same isolated margin that liquidation and ADL consume. The ADL planner already uses funding-adjusted balances; keep that and size maxFunding so one window cannot consume more margin than the liquidation and backstop waterfall can process between hourly settle-all passes.

Invariant Implementation rule
Funding cannot hide debt. Risk-increasing actions must apply on-chain funding or simulate the same index debit before admission.
Funding settlement cannot block repair. Funding settlement should not revert for underfunded accounts; it should floor margin, emit write-off telemetry, and route the account to liquidation / ADL.
ADL takers are evaluated after funding. Planner ranking must continue to use funding-adjusted balances, matching _settlePendingFunding.
Closed-session accrual cannot surprise reopen. Equity-market funding status changes must settle the current index to the boundary or fail closed; never silently skip settlement.

Implementation path

1. Shadow sample

Add an HTV1 shadow sampler that records one impact bid/ask sample per minute. Compare its window output against today's recordTrade clearing-price output for at least two weeks before switching production funding.

2. Swap input

Replace or extend FundingOracle.recordTrade with an impact-sample entrypoint. Keep Perpetual.setFundingRate() ordering: _loadContext(true) first, then compute the next per-second rate.

3. Operate

Keep funding-rate-updater, updateGlobalIndex() refreshes, and funding-settlement-keeper hourly settle-all batches. Make setOffChainFundingRate alerting and duration limits mandatory for emergency use.

Launch blocker: the current after-hours funding disable path may emit FundingStatusSettlementSkipped if index refresh reverts, discarding unsettled accrual. The replacement must settle to the session boundary or fail closed before toggling accrual.
